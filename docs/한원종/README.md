---

### 인사

```
안녕하십니까?

저희는 마음을 녹일 길냥이 서비스 모에모에냥입니다.

저는 모에모에냥에서 발표를 맡은 한원종입니다.

반갑습니다.
```

### 목차

```
발표 순서는 다음과 같습니다.

우선 기획 의도와 기능에 대해서 설명한 후, 
서비스 목적을 간단하게 짚고 넘어가겠습니다.

그리고 프론트엔드 기획을 말씀드리고,
이어서 아키텍처와 백엔드 기획에 대해서 짚고 넘어가겠습니다.

그리고 기능 명세서와 API 명세서에 대해 간단하게 설명한 후,
팀원 소개로 발표를 마무리하고,
이 후 질의응답을 진행하는 순서로 진행하겠습니다.
```

## 기획 의도

---

### 인공 지능 도메인으로서의 기획 의도

```
먼저 기획 의도입니다.

우선 인공 지능 도메인으로서의 기획 의도에 대해 말씀 드리겠습니다.

저희는 머신러닝을 이용하여 
동영상을 이미지 캡셔닝하는 것을 목표로 하였습니다.

그래서 객체의 인식과 검출, 생성 등 다양한 기법을 활용해 볼 수 있는 
'자세 분석 기술'을 사용하고자 하였습니다.

무엇보다 광범위한 주제를 다루기 보다는
특화된 도메인을 선정하고자 하였는데,
그 중에서도 데이터셋이 많은 
동물의 '행동분석'이라는 도메인을 선정하여 아이디어 기획을 시작했습니다.
```

### 길고양이를 선택한 이유

```
저희는 그러한 동물 중에서 길고양이를 선택했는데,
길고양이를 선택한 이유는 다음과 같습니다.

AI Hub의 데이터셋 중에서 '반려동물 행동분석'이라는 데이터가 있었는데, 
특히 강아지와 고양이가 데이터셋이 풍부하여 
머신러닝 학습에 적합하다고 판단했습니다.

그 중에서도 ‘길고양이’를 이용한 서비스가
사용자에게 실외에서의 동적인 경험을 줄 수 있을 것이라 생각했고,
이러한 부분에 매력을 느껴 AI 분석의 대상으로 
길고양이를 선택하게 되었습니다.
```

### 캠퍼스를 선택한 이유

```
저희는 이러한 서비스의 대상으로 대학생을 선택하였습니다.

대학생을 대상으로 선정한 가장 중요한 이유는 
높은 참여를 이끌어 낼 수 있기 때문입니다.

대학생이라는 집단의 특성 상 
다른 집단보다 소속감과 참여도가 높다는 사실은
따로 말씀드리지 않아도 여러분 모두 공감하실 거라 생각합니다.

그리고 생각보다 길고양이에 대해 애정을 갖고 있는 
대학생이 많다는 것을 여러 곳에서 확인할 수 있었습니다.

가톨릭대학교의 캣홀릭, 고려대학교의 고고쉼과 같이
길고양이를 돌보는 동아리가 여러가지 존재한다는 점과,
캠퍼스별로 유명한 고양이가 SNS에서 화제가 되었다는 점은
길고양이에 대한 대학생들 높은 관심을 방증한다고 생각합니다.
```

### 기능 설명

```
이러한 서비스를 위해 저희가 구현할 기능은 크게 4가지입니다.

지도를 통해 고양이의 최근 위치를 확인하는 것과,

인공지능 분석을 통해 고양이의 질병을 확인하는 것,

그리고 인공지능을 통해 고양이의 행동을 분석하고
이를 행동별로 조회하는 것,

마지막으로 고양이에게 먹이를 줄 수 있는 급식소를 관리하는 것입니다.
```

### 서비스 목적

```
그래서 저희가 준비한 서비스, 모에모에냥의 목적은
대학교 내에서 생활하는 길고양이들을 대상으로 
대학생들이 쉽게 돌봄을 제공할 수 있도록 하는 것입니다.
```

## 기술 관련

---

### 프론트엔드 기획

```
그래서 저희의 서비스가 사용자에게 잘 다가갈 수 있도록 
프론트엔드를 다음과 같이 기획했습니다.

모바일 환경에 맞게 Swipe, 무한스크롤 등을 활용할 수 있도록 
UI를 구성하였고,

지도 API와 연동하기 위해 Bottom Sheet UI를 활용하였으며,

각 유닛들이 통합되기 용이하도록 테마 색상을 활용하였습니다.
```

### 목업 및 기능 동영상 시연

```
그리고 동영상으로 간단하게 목업 디자인과 기능을 보여드리겠습니다.

먼저 회원가입과 로그인 및 비밀번호 재설정 화면인데,
보시는 대로 회원가입, 로그인, 비밀번호 재설정 기능을 구현하는 페이지입니다.

그리고 메인화면에서는 고양이에 대한 피드를 확인할 수 있습니다.
특이사항이 있을 경우 관리자에게 신고를 할 수 있고,
인공지능으로 분석된 동작 태그 별로 게시물을 확인할 수 있습니다.

그리고 대학별로 등록된 고양이 목록을 확인할 수 있는데,
여기서 고양이를 등록할 수도 있습니다.

그리고 각 고양이별 상세 페이지에서는
고양이에 대한 팔로우 기능과 최근 위치,
그리고 고양이의 사진과 영상을 확인할 수 있습니다.

그리고 질병이 있는 고양이라면 질병 타임라인에서
해당 고양이가 어떤 질병을 가지고 있는지 확인할 수 있습니다.

이러한 질병은 인공지능 분석을 통해 확인할 수 있습니다.

만약 분석 결과에 질병이 없다고 나왔음에도 불구하고,
질병으로 의심되는 경우에는 지도에서 근처의 병원을 확인할 수 있습니다.

그리고 사용자 프로필 페이지에서는 업적에 따라 부여된 배지와
현재 팔로우중인 고양이의 리스트를 확인할 수 있습니다.

그리고 지도에서는 고양이의 최근 위치와 급식소의 위치를 확인할 수 있는데,
토글로 급식소의 마커를 켜고 끌 수 있습니다.

고양이의 마커를 클릭한다면 메인화면처럼 고양이의 피드를 확인할 수 있고,
급식소 마커를 클릭한다면 해당 급식소의 정보를 확인할 수 있습니다.

급식소에서는 밥줬어요 버튼을 통해 급식소에 고양이들이 먹을 수 있는
밥을 채웠다는 표시를 할 수 있습니다.

그리고 이러한 급식소 역시 사용자가 직접 등록할 수 있도록 하여,
사용자의 참여를 이끌어 낼 수 있습니다.
```

```
프론트엔드 기술스택은 다음과 같습니다.

재사용 가능한 UI 구성 요소 제작과 유지보수를 위해 리액트를 사용하고,

그리고 디자인 시스템 프레임워크 컴포넌트의 디자인을 확인 및 테스트하고 
코드의 재사용성을 높이기 위해 스토리북을 사용하고,

커스터마이징과 용량 최적화 및 
Storybook과 호환성을 위해 테일윈드를 사용합니다.
```

### ERD 설계

