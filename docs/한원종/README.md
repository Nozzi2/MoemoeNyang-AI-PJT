    - 발표 내용
      
      - **기획 의도, 기능 명세서, 피그마(UI 명세서), ERD, API 명세서, 기술 스택**
      
      프로젝트 발표를 효과적으로 전달하려면 다음과 같은 절차를 따를 것을 추천합니다.
      
      1. 기획 의도 설명: 프로젝트의 목적과 배경, 문제 해결 방안 등을 설명합니다. 이 단계에서는 왜 이 프로젝트가 필요한지, 어떤 문제를 해결하고자 하는지에 대해 자세히 설명합니다.
      2. 기능 명세서 소개: 프로젝트에서 구현한 기능에 대한 명세서를 간략하게 소개합니다. 이 단계에서는 프로젝트가 어떤 기능을 제공하며, 사용자에게 어떤 이점을 제공하는지에 대해 설명합니다.
      3. 피그마(UI 명세서) 소개: 프로젝트에서 구현한 UI 디자인을 보여줍니다. 이 단계에서는 프로젝트의 UI가 어떻게 구성되어 있는지, 어떤 디자인 원칙을 적용했는지에 대해 설명합니다.
      4. ERD 소개: 프로젝트에서 사용한 데이터베이스 스키마를 보여줍니다. 이 단계에서는 데이터베이스 모델링에 대한 이해가 필요하며, 어떤 테이블과 컬럼으로 구성되어 있는지에 대해 설명합니다.
      5. API 명세서 소개: 프로젝트에서 구현한 API에 대한 명세서를 간략하게 소개합니다. 이 단계에서는 프로젝트에서 어떤 API를 제공하며, 어떻게 사용할 수 있는지에 대해 설명합니다.
      6. 기술 스택 소개: 프로젝트에서 사용한 기술 스택을 소개합니다. 이 단계에서는 프로젝트를 구현하는 데 사용한 프로그래밍 언어, 프레임워크, 라이브러리 등에 대해 설명합니다.
      
      위의 단계들을 시각적인 자료와 함께 보여주어 시각적인 효과를 높이고, 간결하고 명확하게 발표하는 것이 좋습니다. 또한, 발표 도중에 참가자들이 질문을 할 수 있도록 발표 후 질의응답 시간을 가지는 것이 도움이 됩니다.

    동물의 행동분석이라는 특화된 도메인으로 선정하여 아이디어 기획을 시작했습니다.

    ### 길고양이를 선택한 이유
    
    ```
    수 많은 동물 중에 그 중에 길고양이를 선택한 이유는 다음과 같습니다.
    
    AI Hub의 데이터셋 중 반려동물 행동분석 데이터가 있었는데, 
    특히 강아지와 고양이는 데이터셋이 풍부하여 머신러닝 학습에 적합합니다.
    
    이 중 ‘길고양이’를 이용한 서비스가 사용자에게 
    실외에서의 동적인 사용자 경험을 줄 수 있을 것이라 생각했습니다.
    ```
    
    ### 캠퍼스를 선택한 이유
    
    ```
    그렇다면 이러한 서비스의 대상으로 대학생을 선택한 이유는 
    높은 참여를 이끌어 낼 수 있기 때문입니다.
    
    에브리타임을 이용해 보신 분들은 아시겠지만,
    대학생은 타 집단보다 소속감과 참여도가 높습니다.
    
    그리고 고려대학교의 고고쉼, 가톨릭대학교의 캣홀릭 등 
    고양이들을 돌보는 동아리도 여러개 존재합니다.
    
    무엇보다 SNS에서 화제가 된 고양이들처럼 대학교때 캠퍼스 마스코트 고양이가 있는데,
    이렇게 길고양이를 위한 동아리가 존재하는 점, 사람들에게 불리는 길고양이의 애칭이 있다는 사실은
    길고양이에 대한 대학생들 높은 관심을 방증한다고 생각합니다.
    ```
    
    ### 서비스 목적
    
    ```
    그래서 저희 서비스의 목적은
    대학교 내에서 생활하는 길고양이들을 대상으로 
    대학생들이 쉽게 돌봄을 제공할 수 있도록 하는 것입니다.
    
    따라서 이러한 서비스가 제공하는 기능을 간략하게 설명하면 다음과 같습니다.
    
    고양이 사진과, 영상을 게시하여 사용자끼리 고양이의 귀여운 사진과 영상을 공유할 수 있습니다.
    
    이러한 사진과 영상은 인공지능을 통해 행동과 질병을 분석할 수 있습니다.
    
    지도에 표시된 대학교 캠퍼스 내의 급식소를 관리할 수 있습니다.
    
    그리고 지도에 근처의 동물병원을 표시하여 위급한 상황에 대비할 수 있습니다.
    
    구체적인 구현 방식은 추후 목업 이미지를 보면서 설명해 드리겠습니다.
    ```
    
    ### 팀원 역할 소개
    
    ```
    이러한 서비스 구현을 위한 팀원을 소개하자면 다음과 같습니다.
    
    백엔드는 
    팀장, 노정환 교육생
    송승현 교육생
    이영준 교육생
    을 포함하여 총 3명으로 구성되어 있습니다.
    
    송수현 교육생
    최종현 교육생
    프론트엔드와 발표를 맡은 한원종 교육생
    을 포함하여 총 3명으로 구성되어 있습니다.
    ```
